# Лендинг-портфолио — Рамазан Оразханов, таргетолог

Одностраничный статический сайт-портфолио. Без бэкенда, без сборщиков и npm.
Весь код — в одном файле `index.html` (CSS и JS встроены inline). Готов к публикации
на GitHub Pages. Будущий домен: **orazkhanov.kz**.

## Структура проекта

```
Референсы/
├── index.html        ← весь сайт (HTML + CSS + JS в одном файле)
├── assets/           ← картинки
│   └── roma.jpg      ← фото для hero и og:image (превью ссылок)
├── README.md         ← этот файл
├── server.py         ← локальный сервер для предпросмотра (нужен только Claude Code)

```

> `server.py` и `.claude/` — вспомогательные, на сам сайт не влияют. На GitHub можно
> их не загружать (см. `.gitignore` ниже).

## Ассеты (папка `assets/`)

Сайт работает даже без картинок — код аккуратно это обрабатывает:

| Файл | Назначение | Если файла нет |
|------|-----------|----------------|
| `roma.jpg` | фото в hero + `og:image` | показывается заглушка «Фото Рамазана» |
| `review-vlad.jpg` | скрин отзыва Владислава (опц.) | блок скрина просто исчезает |
| `case-makbuh.jpg` | скрин кабинета MakBuh (опц.) | остаются только цифры кейса |
| `case-webtours.jpg` | скрин кабинета Web Tours (опц.) | остаются только цифры кейса |
| `case-holst.jpg` | скрин Holst.kz (опц.) | остаются только цифры кейса |

(регистр важен, расширение — `.jpg`). Обновите страницу — картинка появится сама.

## Локальный просмотр

Самый простой способ — открыть `index.html` двойным кликом в браузере.

Либо поднять локальный сервер (тогда корректно грузятся картинки из `assets/`):

```bash
cd "путь/к/папке/Референсы"
python3 -m http.server 4599
# затем открыть http://localhost:4599
```

## Деплой на GitHub Pages

### 1. Создать репозиторий
1. Зайдите на https://github.com → **New repository**.
2. Имя — любое (например, `landing` или `orazkhanov-kz`), тип **Public**.
3. Создайте репозиторий (пока пустой).

### 2. Загрузить файлы
Через сайт GitHub: на странице репозитория **Add file → Upload files** →
перетащите `index.html` и папку `assets/` → **Commit changes**.

Или через терминал:
```bash
cd "путь/к/папке/Референсы"
git init
git add index.html assets README.md
git commit -m "Лендинг-портфолио"
git branch -M main
git remote add origin https://github.com/ВАШ_ЛОГИН/ИМЯ_РЕПО.git
git push -u origin main
```

### 3. Включить Pages
1. В репозитории: **Settings → Pages**.
2. **Source** → `Deploy from a branch`.
3. **Branch** → `main`, папка `/ (root)` → **Save**.
4. Через 1–2 минуты сайт будет доступен по адресу
   `https://ВАШ_ЛОГИН.github.io/ИМЯ_РЕПО/`.

### 4. Подключить домен orazkhanov.kz
1. **Settings → Pages → Custom domain** → введите `orazkhanov.kz` → **Save**.
   (GitHub сам создаст в репозитории файл `CNAME`.)
2. В панели вашего регистратора домена (там, где куплен `.kz`) добавьте DNS-записи:

   **Вариант A — домен без www (apex):** четыре A-записи на IP GitHub Pages:
   ```
   A   @   185.199.108.153
   A   @   185.199.109.153
   A   @   185.199.110.153
   A   @   185.199.111.153
   ```
   **Вариант B — поддомен www:** одна CNAME-запись:
   ```
   CNAME   www   ВАШ_ЛОГИН.github.io.
   ```
   (Рекомендуется настроить и apex, и www.)
3. Вернитесь в **Settings → Pages**, дождитесь проверки домена и включите
   **Enforce HTTPS** (галочка появится после того, как DNS «прорастёт» — обычно
   от 10 минут до нескольких часов).

Готово — сайт открывается по `https://orazkhanov.kz`.
